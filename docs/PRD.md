# PRD: 삼루만 타이머 (SamrumanTimer)

## 개요

**삼루만 타이머**는 치지직(CHZZK) 스트리머를 위한 **방송 연장 컨텐츠용 카운트다운 타이머 서비스**다. 스트리머가 "방송 종료까지 남은 시간" 타이머를 OBS에 띄워놓고, 시청자의 요청(후원 등)에 따라 스트리머가 수동으로 시간을 추가/차감하는 방식으로 운영한다. 변경 이력을 시각화하여 방송 콘텐츠로 활용할 수 있다.

## 핵심 기능

### 1. 프로젝트 관리
- 스트리머(소유자)가 프로젝트를 생성하고 관리
- 프로젝트 단위로 타이머를 그룹화
- 프로젝트 목록은 누구나 열람 가능

### 2. 타이머
- 프로젝트당 하나의 타이머 사용 (1 프로젝트 = 1 타이머)
- 카운트다운 방식: 방송 종료까지 남은 시간이 실시간으로 감소
- 상태: RUNNING(진행 중) / EXPIRED(만료)
- 시간 증감: 소유자(스트리머)가 시청자 요청(후원 등)에 따라 수동 입력. 프리셋(+1h, +5h, +10h) 및 직접 입력 지원
- 만료 시 자동 상태 전환
- 만료된 타이머에 시간 추가 시 자동 재오픈

### 3. 로그
- 타이머의 모든 변경 사항을 기록
- 액션 타입: ADD(시간 추가), SUBTRACT(시간 차감), CREATE(생성), EXPIRE(만료), REOPEN(재오픈)
- 자동 감소(카운트다운)는 로깅하지 않음
- 변경자 이름(시간 변경을 요청한 시청자의 닉네임), 변경 전/후 시간, 변경량 기록
- 페이지네이션 지원

### 4. 그래프 시각화
- Recharts 라이브러리 사용
- 3가지 모드:
  - **잔여 시간 추이**: 시간에 따른 잔여 시간 변화 (LineChart)
  - **누적 변경량**: 누적 추가/차감량 (AreaChart)
  - **이벤트 빈도**: 시간대별 이벤트 발생 횟수 (BarChart)

### 5. 인증
- CHZZK OAuth를 통한 로그인
- 쿠키 기반 JWT 세션 관리
- 비로그인 사용자: 조회만 가능
- 로그인 사용자(소유자): 프로젝트/타이머 생성, 시간 조작

## 사용자 시나리오

### 시나리오 1: 방송 시작 전 타이머 준비
1. 스트리머가 CHZZK 계정으로 로그인
2. 새 프로젝트 생성 (예: "24시간 게임 마라톤")
3. 프로젝트에 타이머 생성 (예: 초기 2시간, 예약 시작 가능)
4. 타이머 상세 페이지 URL을 OBS 브라우저 소스에 등록
5. 방송 시작 시 타이머가 OBS 화면에 카운트다운으로 표시됨

### 시나리오 2: 방송 중 시청자 요청으로 시간 추가/차감
1. 시청자가 후원(도네이션)과 함께 "1시간 추가해주세요!" 요청
2. 스트리머가 타이머 상세 페이지에서 프리셋 버튼 또는 직접 입력으로 시간 추가
3. 시청자 닉네임을 입력 (누가 요청했는지 기록)
4. 변경 즉시 반영 — OBS 화면의 카운트다운에 실시간 반영
5. 로그에 "시청자A가 1시간 추가" 기록

### 시나리오 3: 시청자가 OBS 화면 또는 웹에서 타이머 확인
1. 시청자는 방송 화면(OBS 오버레이)에서 실시간 카운트다운 확인
2. 또는 타이머 페이지 링크로 직접 접속하여 확인
3. 로그와 그래프로 누가 얼마나 시간을 추가/차감했는지 이력 확인

## OBS 활용

- 타이머 상세 페이지(`/timers/[id]`)를 OBS 브라우저 소스로 등록하여 방송 화면에 카운트다운 오버레이로 사용
- 시청자는 방송 화면에서 실시간으로 남은 시간을 확인할 수 있음
- 스트리머는 별도 브라우저 창에서 타이머 조작 (시간 추가/차감)

## 기술 스택
- **프론트엔드**: Next.js 16 (App Router), React 19, TypeScript, Tailwind CSS v4
- **차트**: Recharts
- **백엔드**: Next.js API Routes
- **데이터베이스**: Cloudflare D1 (SQLite)
- **인증**: CHZZK OAuth + JWT (httpOnly 쿠키)
- **배포**: Cloudflare Pages (@opennextjs/cloudflare)

## 비기능 요구사항
- 타이머 잔여 시간은 서버에서 계산 (클라이언트 시계 의존 X)
- 실시간 UI 업데이트는 클라이언트 setInterval로 표시만 담당
- 페이지 새로고침 시 서버 값으로 동기화
- 모바일 반응형 지원
