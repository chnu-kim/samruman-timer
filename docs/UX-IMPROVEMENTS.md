# UX 개선 기획서

> 삼루만타이머 UX 개선 기능 기획서. 각 항목은 체크리스트로 진행 상황을 추적한다.

---

## 시장 환경 요약

방송용 타이머 도구 시장의 2024-2026년 핵심 트렌드:

1. **자동화 통합** — 후원/구독 이벤트 시 자동 시간 추가가 업계 표준 (StreamElements, StreamTimer.io, Subathon Timer)
2. **멀티플랫폼** — Twitch, YouTube, TikTok, Kick 등 복수 플랫폼 동시 지원 필수화. 삼루만타이머는 치지직(CHZZK) 전용 니치 전략
3. **시각적 커스터마이징 고도화** — 프리미엄 테마/애니메이션/폰트 선택이 차별화 포인트 (Streamlabs: 20+종 폰트, 펄스/플립 애니메이션)
4. **채팅 커맨드** — `!addtime`, `!pause` 등 채팅 기반 제어가 시청자 참여의 핵심 수단
5. **실시간 제어 패널** — 별도 웹 대시보드/Stream Deck 통합 보편화

## 기능 제안

### P0: 즉시 — 핵심 UX 안정화

- [x] **P0-1. 타이머 상세 폴링 자동화**
  - 사용자 스토리: 스트리머로서 타이머 페이지가 항상 최신 상태이길 원한다, 수동 새로고침 없이
  - 예상 공수: 0.5일
  - 현재 문제: `fetchTimer`가 최초 1회만 호출되고 서버 동기화가 `handleModified`에만 의존. 소유자가 아닌 시청자가 볼 때 변경 사항 미반영
  - 개선안:
    - RUNNING 상태: 5초 간격 폴링으로 서버 잔여 시간 동기화
    - EXPIRED/SCHEDULED 상태: 15초 간격으로 상태 전환 감지
    - 탭 비활성화 시 폴링 중단 (Page Visibility API)
    - 클라이언트-서버 차이 2초 이상이면 서버 값으로 보정
  - 구현 위치: `src/app/timers/[id]/page.tsx`
  - 기술 제약: Cloudflare Workers CPU 시간 제약상 5초 미만 불가. SSE/WebSocket은 Workers에서 불가하므로 폴링이 최선

- [ ] **P0-2. Optimistic UI 적용**
  - 사용자 스토리: 스트리머로서 시간 추가 시 즉각적인 피드백을 보고 싶다, 대기 없이 빠르게 연속 조작하기 위해
  - 예상 공수: 1일
  - 현재 문제: `submitModify` 호출 시 API 응답 대기 후 UI 갱신. 빠른 연속 조작 시 체감 지연 발생
  - 개선안:
    - `submitModify` 호출 즉시 `onModified`를 optimistic하게 호출하여 카운트다운 먼저 갱신
    - API 실패 시 이전 값으로 롤백 + 에러 토스트
    - 연속 입력 시 마지막 응답으로 서버 값 확정
  - 구현 위치: `src/components/timer/TimerControls.tsx`

- [ ] **P0-3. 타이머 만료 임박 시각적 경고**
  - 사용자 스토리: 스트리머로서 타이머가 곧 만료될 때 눈에 띄는 경고를 보고 싶다, 만료를 놓치지 않기 위해
  - 예상 공수: 0.5일
  - 개선안:
    - 잔여 5분 미만: `text-amber-500` + 느린 pulse 애니메이션
    - 잔여 1분 미만: `text-red-500` + 빠른 pulse 애니메이션
    - 오버레이에도 동일 적용 (쿼리 파라미터 `urgentColor` 지원)
    - 접근성: `aria-label`에 "긴급" 문구 추가
  - 구현 위치: `src/components/timer/CountdownDisplay.tsx`

### P1: 단기 — 시각적 만족도 및 편의성

- [ ] **P1-1. 최근 사용 닉네임 UI 개선 (드롭다운)**
  - 사용자 스토리: 스트리머로서 자주 후원하는 시청자 닉네임을 빠르게 선택하고 싶다, 반복 입력을 줄이기 위해
  - 예상 공수: 1일
  - 개선안:
    - `<datalist>` 대신 커스텀 드롭다운 컴포넌트로 교체
    - 최근 닉네임을 칩(chip) 형태로 입력 필드 아래에 표시
    - 칩 클릭 시 즉시 닉네임 필드에 입력
    - "기본 닉네임" 상태를 별 아이콘으로 시각적 구분
  - 구현 위치: `src/components/timer/TimerControls.tsx`

- [ ] **P1-2. 오버레이 애니메이션 효과**
  - 사용자 스토리: 스트리머로서 시간 추가 시 오버레이에 시각 효과를 보여주고 싶다, 시청자 참여를 시각화하기 위해
  - 예상 공수: 2일
  - 개선안:
    - 시간 추가 시: 숫자가 초록색으로 flash + scale-up 후 원복
    - 시간 차감 시: 빨간색으로 flash + shake 효과
    - `+1:00:00` 같은 변경량 텍스트를 floating으로 표시 후 fade-out
    - 오버레이 설정에 "애니메이션 켜기/끄기" 토글 추가
    - CSS animation만 사용 (JS 의존 최소화)
  - 구현 위치: `src/app/timers/[id]/overlay/page.tsx`, `src/components/timer/OverlaySettings.tsx`

- [ ] **P1-3. 사운드 알림 (시간 추가/만료)**
  - 사용자 스토리: 스트리머로서 시간이 추가되거나 만료될 때 효과음을 듣고 싶다, 다른 화면을 보고 있어도 인지하기 위해
  - 예상 공수: 1일
  - 개선안:
    - 시간 추가 시 짧은 "coin" 효과음
    - 타이머 만료 시 알람 효과음
    - 타이머 상세 페이지에 음소거 토글 아이콘 (기본: 음소거)
    - 오버레이에는 사운드 미적용 (OBS에서 별도 설정)
    - Web Audio API 사용, `/public/sounds/`에 10KB 이하 .mp3 배치
  - 기술 제약: Cloudflare Pages 정적 에셋 크기 제한 고려

- [ ] **P1-4. 오버레이 프리셋 확장 + 커스텀 CSS**
  - 사용자 스토리: 스트리머로서 내 방송 브랜딩에 맞는 오버레이를 만들고 싶다, 전문적인 느낌을 주기 위해
  - 예상 공수: 1.5일
  - 구현 위치: `src/components/timer/OverlaySettings.tsx`

### P2: 중기 — 기능 확장

- [ ] **P2-1. 공유용 하이라이트 페이지**
  - 사용자 스토리: 스트리머로서 마라톤 방송 결과를 SNS에 공유하고 싶다, 성과를 시청자와 나누기 위해
  - 예상 공수: 2일
  - 개선안:
    - `/projects/[id]/summary` 경로
    - 마라톤 결과 요약: 총 운영 시간, 총 추가 시간, 참여 시청자 수, TOP 3 후원자
    - OG 메타 태그 자동 생성
    - SNS 공유 버튼 (링크 복사, Twitter/X 공유 텍스트 생성)
    - 서버 컴포넌트로 구현 (SEO 최적화)

- [ ] **P2-2. 타이머 일시정지(PAUSE) 상태**
  - 사용자 스토리: 스트리머로서 방송 중 잠시 타이머를 멈추고 싶다, 휴식 시간에 시간이 소모되지 않도록
  - 예상 공수: 2일
  - 개선안:
    - 새 상태: `PAUSED` (RUNNING → PAUSED → RUNNING 전환)
    - 일시정지 시 `baseRemainingSeconds`를 현재 잔여 시간으로 확정
    - 로그 액션 타입: `PAUSE`, `RESUME` 추가
    - UI: TimerControls에 재생/일시정지 토글 버튼
    - 오버레이: 일시정지 시 "PAUSED" 텍스트 오버레이
  - 기술 영향: DB 마이그레이션 필요 (`TimerStatus` 확장), `timer.ts` 계산 로직 수정

- [ ] **P2-3. 실시간 시청자 뷰 전용 페이지**
  - 사용자 스토리: 시청자로서 깔끔한 전용 페이지에서 타이머와 로그를 보고 싶다, 방송을 안 켜고도 확인하기 위해
  - 예상 공수: 1.5일

### P3: 장기 — 비전/차별화

- [ ] **P3-1. 오버레이 위젯 다양화 (진행바, 목표 표시)**
  - 사용자 스토리: 스트리머로서 오버레이에 목표 달성률 바를 표시하고 싶다, 시청자 참여 동기를 높이기 위해
  - 예상 공수: 3일

- [ ] **P3-2. 프로젝트 템플릿**
  - 사용자 스토리: 스트리머로서 이전 마라톤 설정을 재사용하고 싶다, 매번 새로 설정하는 번거로움을 줄이기 위해
  - 예상 공수: 1.5일

## 기능 제거 제안

> 핵심 흐름(타이머 생성 → 시간 조절 → OBS 표시)에 영향 없이 정보 과부하를 줄인다.

- [x] **프로젝트/타이머 통계 중복 제거**
  - 현재 위치: `/projects/[id]/stats`, `/timers/[id]/stats`
  - 제거 사유: 프로젝트당 타이머 1개(1:1 관계)이므로 두 통계 페이지가 사실상 동일 데이터. 유지보수 비용 2배
  - 대안: 타이머 통계 페이지만 유지, 프로젝트 상세에서 타이머 통계로 직접 링크
  - 영향도: 낮음

- [ ] **빠른 적용 모드 토글 제거**
  - 현재 위치: `TimerControls.tsx` 내 `quickMode`
  - 제거 사유: 일반 모드와의 차이가 직관적이지 않아 인지 부하 증가. 모바일에서는 하단 고정 바로 이미 빠른 적용 가능
  - 대안: 프리셋 버튼을 "탭하면 누적, 길게 누르면 즉시 적용"으로 통합하거나, 닉네임 설정 상태에서는 항상 즉시 적용
  - 영향도: 중간

- [ ] **그래프 모드 과잉 정리**
  - 현재 위치: `GraphModeSelector` 3종
  - 제거 사유: "이벤트 빈도" 그래프가 통계 페이지 `HourlyActivityChart`와 기능 중복. 타이머 상세에 3개 그래프 모드는 페이지를 무겁게 만듦
  - 대안: 타이머 상세에는 "잔여 시간 추이"만 유지, "누적 변경량"과 "이벤트 빈도"는 통계 페이지로 이동
  - 영향도: 중간
