# 기획 규칙

기능 관련 논의 시 모든 에이전트가 참조하는 규칙.

## 기능 제안 원칙

- 기능 제안 시 시장 근거(데이터/통계/경쟁사 사례) 필수
- `docs/PRD.md`와의 정합성 확인 필수
- 사용자 스토리 형식 필수: "~로서 ~하고 싶다, ~하기 위해"
- 성공 지표는 측정 가능한 KPI로 정량화

## 기술 실현 가능성

- 현재 스택: Next.js 16, D1 (SQLite), Cloudflare Pages, React 19, Tailwind CSS v4
- 스택 변경이 필요한 기능은 변경 비용/리스크를 명시
- Cloudflare Workers 제약(CPU 시간, 메모리, 실행 시간)을 고려
- D1 SQLite 제약(트랜잭션, 동시성, 데이터 타입)을 고려

## 우선순위 체계

- **P0 (즉시)**: 사용자 이탈 방지, 핵심 사용성 개선
- **P1 (단기)**: 성장/바이럴에 직접 기여
- **P2 (중기)**: 경쟁 우위 확보
- **P3 (장기)**: 비전/차별화

## 출력 언어

- 기획서, 사용자 스토리, 분석 결과: 한국어
- 기술 용어, 코드 관련 명칭: 영어 유지
