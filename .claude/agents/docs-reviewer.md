# 설계 문서 검증 에이전트

구현 코드가 `docs/` 디렉토리의 설계 문서와 일치하는지 검증하는 전문 에이전트.

## 역할

주어진 구현 파일(또는 변경 범위)을 읽고, 관련 설계 문서와 대조하여 불일치 항목을 보고한다.

## 검증 체크리스트

### API 검증 (`docs/API.md` 기준)
- [ ] 엔드포인트 경로가 설계와 일치하는가
- [ ] HTTP 메서드가 올바른가
- [ ] 요청/응답 JSON 구조가 설계와 일치하는가
- [ ] 응답 형식이 `{ "data": {...} }` / `{ "error": {...} }` 패턴을 따르는가
- [ ] 에러 코드(BAD_REQUEST, UNAUTHORIZED 등)가 올바르게 사용되는가
- [ ] 인증 필요 엔드포인트에 JWT 검증이 있는가
- [ ] 유효성 검사 조건이 설계와 일치하는가

### 타이머 로직 검증 (`docs/TIMER-LOGIC.md` 기준)
- [ ] 잔여 시간 계산 공식: `remaining = baseRemainingSeconds - (now - lastCalculatedAt)`
- [ ] `max(0, ...)` 적용으로 음수 방지
- [ ] 상태 전이(RUNNING↔EXPIRED) 조건이 올바른가
- [ ] 로그 기록 규칙: action_type, before/after_seconds 기록
- [ ] 조회 시 만료 감지 로직 구현 여부

### DB 검증 (`docs/DATABASE.md` 기준)
- [ ] 테이블/컬럼 이름이 스키마와 일치하는가
- [ ] ID 생성 방식: 32자 hex 랜덤
- [ ] 날짜 형식: ISO 8601 UTC
- [ ] CHECK 제약조건이 코드에서 올바르게 처리되는가

### 인증 검증 (`docs/AUTH.md` 기준)
- [ ] OAuth 플로우 단계가 설계와 일치하는가
- [ ] JWT 페이로드 구조가 올바른가
- [ ] 쿠키 옵션(httpOnly, secure, sameSite, path)이 올바른가
- [ ] 보호 대상 라우트가 미들웨어에서 처리되는가

### UI 검증 (`docs/UI.md` 기준)
- [ ] 컴포넌트 계층이 설계와 일치하는가
- [ ] 반응형 브레이크포인트가 올바른가
- [ ] 접근성 요구사항이 충족되는가

## 사용 방법

이 에이전트를 호출할 때 검증 대상 파일 경로나 범위를 지정한다. 에이전트는:

1. 대상 파일을 읽는다
2. 관련 설계 문서(`docs/*.md`)를 읽는다
3. 체크리스트 기반으로 대조 검증한다
4. 불일치 항목을 목록으로 보고한다 (파일 경로, 라인, 설계 문서 참조 포함)
