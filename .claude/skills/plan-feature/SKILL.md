# 데이터 기반 기능 기획

시장 데이터, 경쟁사 분석, 사용자 행동 통계를 기반으로 새로운 기능을 기획한다.

## 입력

호출 시 다음 중 하나를 지정한다:

- 영역: `timer`, `project`, `social`, `analytics`, `monetization`, `ux`
- 주제: 자유 텍스트 (예: "시청자 참여 기능", "모바일 경험 개선")
- `all` — 전체 프로젝트 분석 후 기능 제안

## 분석 카테고리

### 1. 시장 환경 분석
- 스트리밍 시장 규모, 치지직/트위치/유튜브 라이브 트렌드

### 2. 경쟁사/유사 서비스 분석
- StreamElements, Streamlabs, OBS 플러그인 기능 비교

### 3. 사용자 행동 분석
- 스트리머 워크플로우, 시청자 참여 메커니즘, 후원 행동 통계

### 4. 현재 기능 갭 분석
- PRD 대비 미충족 니즈, 사용자 시나리오 확장 가능성

### 5. 수익화/성장 전략
- 프리미엄 기능, 바이럴 메커니즘, 리텐션

### 6. 기술 실현 가능성
- 현재 스택(Next.js 16, D1, Cloudflare Pages)으로 구현 가능 여부

### 7. UX 개선 기회
- 모바일 경험, 실시간 동기화, 알림, 접근성, 국제화

### 8. 데이터/분석 기능
- 스트리머 대시보드, 방송 통계, 시청자 참여 인사이트

## 우선순위 기준

- **P0 (즉시)**: 사용자 이탈 방지, 핵심 사용성 개선
- **P1 (단기)**: 성장/바이럴에 직접 기여
- **P2 (중기)**: 경쟁 우위 확보
- **P3 (장기)**: 비전/차별화

## 동작 순서

**반드시 `product-planner` 에이전트를 Agent 도구로 호출하여 작업을 위임한다.**

스킬 호출 시 아래와 같이 에이전트에 위임한다:

```
Agent 도구 호출:
  subagent_type: general-purpose (또는 기본)
  description: "기능 기획 [대상]"
  prompt: |
    You are the product-planner agent (.claude/agents/product-planner.md).
    대상: [사용자가 지정한 영역/주제]
    위 에이전트 정의의 워크플로우 6단계를 그대로 수행하라.
    코드 수정은 하지 않는다. 기획서만 출력하라.
```

### 에이전트 위임 규칙

1. 사용자 입력(영역/주제/all)을 그대로 에이전트 프롬프트에 전달한다
2. 에이전트가 반환한 결과(시장 분석 + 경쟁사 비교 + 기능 제안 + 로드맵)를 사용자에게 그대로 출력한다
3. 기획서 출력 후 구현 여부는 사용자가 결정한다

## 출력 형식

에이전트가 반환하는 형식을 그대로 사용한다:

```
## 📋 기능 기획서: [주제]

### 시장 환경 요약
- [WebSearch 기반 데이터 3~5개 bullet]

### 경쟁사 분석
| 서비스 | 핵심 기능 | 차별화 포인트 | 삼루만타이머 대비 |
|--------|----------|---------------|------------------|

### 기능 제안

| # | 기능명 | 우선순위 | 임팩트 | 노력도 | 근거 |
|---|--------|----------|--------|--------|------|
| 1 | ... | P0 | H | S | [데이터 출처] |

### 상세 기획

#### 기능 #1: [기능명]
- **사용자 스토리**: ~로서 ~하고 싶다, ~하기 위해
- **핵심 요구사항**: [bullet list]
- **시장 근거**: [데이터/통계/경쟁사 사례]
- **기술 고려사항**: [현재 스택 기반 실현 방안]
- **성공 지표**: [측정 가능한 KPI]

### 추천 로드맵
- Phase 1 (즉시): [P0 기능들]
- Phase 2 (1~2주): [P1 기능들]
- Phase 3 (1개월): [P2 기능들]
- Phase 4 (장기): [P3 기능들]
```
