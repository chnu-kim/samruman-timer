# UX 개선 구현

`docs/UX-IMPROVEMENTS.md` 기획서의 미완료 항목을 선택하여 구현하고, 완료 시 체크리스트를 갱신한다.

## 동작 순서

1. `docs/UX-IMPROVEMENTS.md` 읽기 → 미완료(`- [ ]`) 항목 파싱
2. 사용자 입력에 따라 구현할 항목 결정
3. 해당 항목의 상세 기획(구현 위치, 개선안) 확인
4. 구현 실행 (코드 수정/추가)
5. 빌드 검증 (`pnpm build`)
6. `docs/UX-IMPROVEMENTS.md`에서 해당 항목 `- [ ]` → `- [x]` 갱신
7. 결과 출력

## 입력 형식

- **항목 ID**: `P0-1`, `P0-2`, `P1-1`, `P2-1` 등 — 해당 항목을 구현
- **`next`**: 가장 높은 우선순위의 미완료 항목을 자동 선택하여 구현
- **`status`**: 전체 진행 상태만 출력 (구현 없이)
- **제거 항목 ID**: `R-1`, `R-2`, `R-3` — 기능 제거 제안의 해당 항목을 실행

### 항목 ID 매핑

**기능 제안:**
- `P0-1` — 타이머 상세 폴링 자동화
- `P0-2` — Optimistic UI 적용
- `P0-3` — 타이머 만료 임박 시각적 경고
- `P1-1` — 최근 사용 닉네임 UI 개선 (드롭다운)
- `P1-2` — 오버레이 애니메이션 효과
- `P1-3` — 사운드 알림 (시간 추가/만료)
- `P1-4` — 오버레이 프리셋 확장 + 커스텀 CSS
- `P2-1` — 공유용 하이라이트 페이지
- `P2-2` — 타이머 일시정지(PAUSE) 상태
- `P2-3` — 실시간 시청자 뷰 전용 페이지
- `P3-1` — 오버레이 위젯 다양화 (진행바, 목표 표시)
- `P3-2` — 프로젝트 템플릿

**기능 제거:**
- `R-1` — 프로젝트/타이머 통계 중복 제거
- `R-2` — 빠른 적용 모드 토글 제거
- `R-3` — 그래프 모드 과잉 정리

## 동작 세부

### 항목 ID 또는 `next` 입력 시

1. `docs/UX-IMPROVEMENTS.md`를 읽고 해당 항목이 미완료(`- [ ]`)인지 확인
2. 이미 완료(`- [x]`)이면 안내 후 종료
3. 항목의 **개선안**, **구현 위치**, **기술 제약** 섹션을 참조하여 구현
4. 구현 완료 후 `pnpm build`로 빌드 검증
5. 빌드 성공 시 `docs/UX-IMPROVEMENTS.md`에서 해당 항목의 `- [ ]`를 `- [x]`로 변경
6. 빌드 실패 시 원인 수정 후 재시도

### `status` 입력 시

1. `docs/UX-IMPROVEMENTS.md`를 읽고 전체 항목 파싱
2. 우선순위별 완료/미완료 현황 테이블 출력
3. 구현은 수행하지 않음

### `next` 우선순위 규칙

P0 미완료 → P1 미완료 → R(제거) 미완료 → P2 미완료 → P3 미완료 순서로 선택.
동일 우선순위 내에서는 번호 순서대로 (예: P0-1 → P0-2 → P0-3).

## 출력 형식

### 구현 완료 시

```
## UX 개선 완료: [항목명]

### 수정 파일
| 파일 | 변경 내용 |
|------|----------|
| `src/...` | ... |

### 빌드 검증
- 결과: PASS / FAIL

### 진행 상황
- 완료: N/M (N%)
- 다음 항목: [다음 미완료 항목명]
```

### `status` 출력 시

```
## UX 개선 진행 상황

| 우선순위 | 전체 | 완료 | 미완료 |
|---------|------|------|-------|
| P0      | 3    | 1    | 2     |
| P1      | 4    | 0    | 4     |
| P2      | 3    | 0    | 3     |
| P3      | 2    | 0    | 2     |
| 제거    | 3    | 0    | 3     |

### 미완료 항목
- [ ] P0-2. Optimistic UI 적용
- [ ] P0-3. 타이머 만료 임박 시각적 경고
- ...
```
